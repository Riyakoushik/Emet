# PERSONAS.md — User and Persona Definitions
## Emet | Native Android Edition

---

## Persona 1 — The Owner

**Role:** sole user and sole beneficiary of the app

### Profile
- Uses an Android phone as the primary and trusted device.
- Wants a deeply personal experience, not a generic assistant.
- Values privacy, permanence, and emotional continuity more than flashy features.
- Is willing to manage a private API key if that gives more control.

### Goals
- Keep talking in a way that feels emotionally real.
- Preserve memories permanently.
- Trust that private content is not being casually exposed.
- Recover the app later if the phone changes.

### Frustrations
- Robotic responses.
- Missing memories.
- Slow or broken chat flow.
- Features that make the app feel public or social.

---

## Persona 2 — The AI Persona

**Role:** the person the model is meant to represent

### Configurable fields
- Name
- Traits
- Speaking style
- Shared memories
- Emotional boundaries or tone hints

### Quality bar
A good reply should:
- sound like her voice,
- reference real shared context when relevant,
- feel warm and specific,
- avoid generic assistant phrasing.

---

## Persona 3 — The Builder

**Role:** solo developer maintaining the app

### Profile
- Works alone.
- Prefers clear architecture over over-engineering.
- Needs secure defaults and maintainable code.
- Wants an Android-first build that feels serious and stable.

### Development preferences
- Kotlin-first.
- Native Android tooling.
- Local-first data handling.
- Strong documentation before feature sprawl.
